# Data Scientist Challenge - Condition Monitoring

Condition monitoring of machinery using vibration signal processing and supervised machine learning.

The challenge consists of two parts:

- **Part 1** вЂ” automatic detection of carpet regions in raw vibration spectra
- **Part 2** вЂ” identification of loose assets from tri-axial vibration signals

---

## Requirements

Python 3.10 or higher is required. Install all dependencies with:

```bash
pip install -r requirements.txt
```

## Part 1 вЂ” Carpet region detection

**1. Run the analysis:**

```bash
python part_1/main_part_1.py
```

**2. Generate figures:**

```bash
python part_1/plot_results_part_1.py
```

**Outputs:**
- `part_1/results/<stem>_spectrum.csv` вЂ” one-sided amplitude spectrum
- `part_1/results/<stem>_results.json` вЂ” metadata, CPR metric, detected regions
- `part_1/results/tables/cpr_table.md` вЂ” all samples ranked by CPR
- `part_1/figures/<stem>_spectrum.png` вЂ” spectrum plot with carpet regions highlighted

**Configuration** (`part_1/config_part_1.json`): detection threshold, frequency bounds, Welch parameters. All values can be overridden at the command line, for example:

```bash
python part_1/main_part_1.py --psd-threshold-db -70 --min-start-freq-hz 500
```

Run `python part_1/main_part_1.py --help` for the full list of options.

## Part 2 вЂ” Structural looseness detection

**1. Run the full pipeline:**

```bash
python part_2/main_part_2.py
```

Or run individual steps in order:

```bash
python part_2/main_part_2.py --run extract       # feature extraction
python part_2/main_part_2.py --run generalize    # nested CV generalization estimate
python part_2/main_part_2.py --run train         # final model training
python part_2/main_part_2.py --run infer         # inference on test data
```

**2. Generate figures:**

```bash
python part_2/plot_results_part_2.py
```

**Outputs:**
- `part_2/results/features.json` вЂ” feature matrix, AUROC, Spearman
- `part_2/results/generalization_metrics.json` вЂ” nested CV performance estimates
- `part_2/results/model.joblib` вЂ” trained model
- `part_2/results/training_info.json` вЂ” best hyperparameters and decision threshold
- `part_2/results/test_results.json` вЂ” predictions, scores, feature contributions
- `part_2/results/spectra/` вЂ” per-sample amplitude spectra (3 orientations)
- `part_2/results/tables/` вЂ” CV metrics and test results as markdown tables
- `part_2/figures/` вЂ” all diagnostic figures

**Configuration** (`part_2/config_part_2.json`): harmonic list, CV folds, random seed, feature extraction parameters.

Run `python part_2/main_part_2.py --help` for the full list of options.
